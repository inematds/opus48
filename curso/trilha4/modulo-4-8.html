A ideia por trás de tudo

Quem está segurando o plano?

Claude normal

Segura o plano inteiro na cabeça. Jobs grandes enchem e desaceleram.

Um Workflow

O plano vive em código e roda sozinho. O Claude só vê o resultado.
É um n8n que desenha o próprio canvas. Você descreve a meta. O Claude desenha o pipeline e roda.

Suba a escada só quando precisar

A resposta padrão é "não é workflow". Suba só quando o degrau de baixo ficar pequeno.
A parte que erram

Os agentes conversam entre si?

Precisa colaborar? Team.   Força bruta paralela? Workflow.

Uma Skill cabe dentro de um Workflow?

Sim. Aninha.
Skill = o como. Workflow = o quantos. Os agentes usam suas skills e apps.

/goal e Workflow são opostos

Seguir até ser verdade? /goal.   Grande demais para uma passada? Workflow.

Isto é poderoso. E um ótimo jeito de queimar dinheiro.

A conta = o tamanho do job que você aponta. Cada agente é uma chamada completa do Claude, e muitos rodam de uma vez.
1. Limite o escopo 2. Nomeie o entregável 3. Braçal no Haiku
Vago → conta alta"Melhore meu projeto inteiro."
Escopado → centavos"Cheque cada arquivo em src/routes por auth faltando. 1 agente Haiku por arquivo. Devolva uma tabela."
Ele mostra o plano antes de rodar. Leia o escopo, depois aprove.

Cadê o código?

Ele escreve um arquivo JavaScript de verdade

export const meta = { name: "audit-skills" } const files = listSkills() await parallel( files.map(f => agent("score " + f)) // 1 agente Haiku por skill ) // → um scorecard ranqueado
O que é→um script JS; um runtime o roda em background enquanto você trabalha.
Salvar→abra /workflows, escolha o run e pressione s.
Reutilizar→vai para .claude/workflows/ e roda como /seu-nome.
Um run é descartável por padrão. Salve e vira um comando. /deep-research é um embutido.
