MÓDULO 4.7

📋 Prompts prontos

Modelos copia-e-cola para disparar workflows com escopo e custo sob controle. Troque os [colchetes] pelos seus dados antes de aprovar o run.

🔎 Auditoria de codebase

Monte um workflow dinâmico que cheque cada arquivo em [src/routes] por [endpoints sem verificação de auth]. Use 1 agente Haiku por arquivo. Não altere nada — só devolva uma tabela: arquivo → risco → linha. Me mostre o escopo antes de rodar.

🧱 Migração em N arquivos

Monte um workflow dinâmico para migrar [todos os arquivos em src/ que usam a API antiga X] para [a nova API Y]. Descubra os arquivos afetados, transforme cada um em isolamento (worktree) e verifique que compila. Modelo Haiku no braçal, Opus só na síntese final. Liste o que mudou por arquivo. Confirme o escopo antes de rodar.

🔬 Deep research escopado

/deep-research [sua pergunta específica, com tema, recorte de tempo e o que comparar]. Priorize fontes [oficiais/primárias]. Saída: [tabela / resumo de 1 página] com citações. Limite a [N] fontes.

🗂️ Auditoria de skills (o do vídeo)

Monte um workflow dinâmico que audite todas as minhas skills (globais e do projeto). 1 agente Haiku por skill, pontuando clareza, front matter (pass/fail) e qualidade do trigger, e apontando o conserto de maior valor. Um agente final (Opus) ranqueia tudo de pior a melhor num scorecard HTML. Me mostre o escopo antes de rodar.

💾 Salvar como comando

Salve este workflow em .claude/workflows/ deste projeto (não no diretório global) com o nome [auditar-skills], para eu poder re-rodar depois como /[auditar-skills].

🛡️ Guard-rails de custo

Antes de rodar: use no máximo [20] agentes, todos em Haiku, escopo restrito a [pasta/lista]. Me mostre o script bruto e o número de agentes estimado para eu aprovar. Pare se passar de [N] agentes.

💡 Regra de ouro dos prompts

Todo prompt de workflow deve carregar as 3 regras: escopo (onde olhar), entregável (o que devolver) e modelo (Haiku no braçal) — mais "me mostre o escopo antes de rodar".
