🧩 A família de orquestração

Skill, Subagent, Agent Team e Workflow se parecem, mas servem para coisas diferentes. A pergunta que separa todos: quantos agentes entram e se eles conversam.

📊 Quatro nomes que se confundem

A forma mais rápida de não confundir é olhar lado a lado. O que muda de um para o outro é quantos agentes entram em cena e se eles se falam.

O que é Quantos agentes Conversam?

💡 A regra de bolso

Uma Skill é uma receita — e pode rodar dentro de qualquer um dos outros três.

2

📒 Skill — a receita

Uma skill é um conjunto reutilizável de instruções que o Claude segue. Não sobe agente extra: o mesmo agente lê e executa. É o "como" de uma tarefa.

  • •Reutilizável: melhora a cada uso
  • •Aninhável: roda dentro de subagent, team ou workflow
  • •Barata: zero overhead de orquestração
3

🧍 Subagent — o ajudante solo

Um subagent é um agente com contexto próprio, enviado para uma tarefa lateral. Mantém a sessão principal limpa, reporta o resultado uma vez e encerra. Não fala com outros subagents — só com a sessão que o chamou.

📊 Quando usar

  • •Uma busca/exploração paralela à conversa principal
  • •Uma tarefa "suja" que você não quer no contexto principal
4

💬 Agent Team × Workflow — eles conversam?

É o ponto que mais gera erro. Agent Team é um grupo de chat: os membros se passam o bastão e reagem entre si. Workflow são pistas solo: os agentes nunca se veem; o que cada um produz só se encontra no merge final.

Precisa colaborar?

Use Agent Team — debate, war room, conselho. Os agentes reagem uns aos outros.

Precisa força bruta paralela?

Use Workflow — muitos agentes independentes, sem conversa, mesclados no fim.

5

🪆 Skill dentro de Workflow — sim, aninha

Os agentes de um workflow podem ler e usar suas skills, seus servidores MCP e suas chaves de API. A skill é o "como"; o workflow é o "quantos".

💡 Reaproveite o que já tem

Tudo que você construiu — skills, MCPs, CLIs — entra dentro de um job massivo sem reescrever nada.

6

🎛️ "Como" × "quantos"

O resumo que organiza a família inteira: a skill define o como uma tarefa é feita; o workflow define quantos e em que largura/profundidade ela roda. Subagent e team ficam no meio.

✓ Bom critério

  • ✓"Quero repetir um procedimento" → Skill
  • ✓"Quero N coisas independentes ao mesmo tempo" → Workflow

✗ Sinais de confusão

  • ✗Usar workflow só porque é novo
  • ✗Esperar que workflow "converse" como team
